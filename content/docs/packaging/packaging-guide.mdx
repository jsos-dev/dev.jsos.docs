# 应用打包

开发完成后，将应用打包为 `.zip` 文件即可分发。

## 打包方式

### 手动打包

```bash
cd my-app/
zip -r ../my-app.zip .
```

### 使用脚本

```bash
# package.sh
APP_DIR=$1
OUTPUT=$2
if [ ! -f "$APP_DIR/package.json" ]; then
  echo "package.json not found"
  exit 1
fi
cd "$APP_DIR"
zip -r "../$OUTPUT" .
cd ..
echo "Created: $OUTPUT"
```

## 打包检查清单

- [ ] `package.json` 在 ZIP **根目录**
- [ ] 包含 `jsos` 字段（`id`、`name`、`startCommand`）
- [ ] 图标使用 data URI 或包内本地文件（不要用网络链接）
- [ ] ZIP 大小合理

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 安装报错"未找到 package.json" | 文件不在 ZIP 根目录 | 确保 `package.json` 在 ZIP 根目录 |
| 图标报错"不支持网络图标" | `icon` 用了 URL | 改为 data URI 或包内文件 |
| 启动时 npm 报错 | 依赖不兼容 | 使用纯 JS 替代包（见注意事项） |
| 服务启动不了 | 端口错误 | 检查是否使用 `process.env.PORT` |

## 注意事项

### 图标不要用网络链接

```json
// ❌ 错误
"icon": "https://example.com/icon.png"

// ✅ 正确
"icon": "data:image/svg+xml,<svg>...</svg>"

// ✅ 正确
"icon": "icon.png"
```

### 包内文件自动挂载

ZIP 内的所有文件会自动挂载到 WebContainer，无需在 `jsos` 中声明 `files` 字段。
