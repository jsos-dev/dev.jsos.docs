# 安装应用

JSOS 支持安装第三方应用，格式为 `.zip` 包。

## 安装方式

### 1. 本地上传（右键菜单）

1. 在桌面空白处**右键点击**
2. 选择 **「安装应用」**
3. 在弹出的文件选择器中选中 `.zip` 文件
4. 系统自动解析并显示应用信息
5. 确认后点击「安装」

### 2. 远程 URL 安装

1. 打开**应用管理器**（App Manager）
2. 选择 **「远程安装」** 选项卡
3. 输入 ZIP 文件的直链地址（如 `https://example.com/app.zip`）
4. 点击「获取」，系统自动下载并解析
5. 确认后点击「安装」

**支持的 URL 格式：**
- `https://example.com/my-app.zip`
- `http://example.com/my-app.zip`

**注意事项：**
- URL 必须是 ZIP 文件的直接下载链接
- 如果遇到跨域问题，可在 **设置 → 代理** 中配置 CORS 代理

### 3. GitHub 安装

1. 打开**应用管理器**（App Manager）
2. 选择 **「GitHub」** 选项卡
3. 输入 GitHub 仓库地址（如 `https://github.com/user/repo`）
4. 点击「获取」，系统自动获取 Release 列表
5. 选择要安装的版本
6. 确认后点击「安装】

**支持的文件名格式：**
- `{appId}-{version}.zip`
- `{repo}-{version}.zip`
- 任何 `.zip` 文件

**注意事项：**
- 无 Release 时自动使用归档下载
- 如果遇到跨域问题，可在 **设置 → 代理** 中配置 CORS 代理

### 4. CLI 命令安装

通过 JSOS 命令行工具安装应用：

```bash
# 安装本地 ZIP 文件
jsos app install ./my-app.zip

# 安装后可查看应用信息
jsos app info my-app
```

**支持的参数：**
- `<file>` — ZIP 文件路径（支持相对路径和绝对路径）

**注意事项：**
- CLI 工具需要在 WebContainer 环境中运行
- 安装成功后应用图标会出现在桌面

## 安装流程

选中 ZIP 包后，系统会做以下检查：

1. **解析 ZIP** — 确认是有效的 ZIP 格式
2. **验证 package.json** — 必须包含 `jsos` 配置字段
3. **检查必要字段** — `id`、`name`、`startCommand` 不能少
4. **版本对比** — 如果已安装同 ID 的应用，会对比版本号
   - 版本更高 → **升级**
   - 版本更低 → **降级**
   - 版本相同 → **覆盖安装**
   - 全新应用 → **安装**

验证通过后显示应用信息弹窗，包含：名称、版本、包含文件数、大小。

## 安装结果

安装过程中会通过 **Toast 通知** 显示进度：

- **成功**：绿色提示，桌面出现应用图标
- **失败**：红色提示，显示具体错误原因

## 常见安装错误

| 错误提示 | 原因 |
|---------|------|
| ZIP 中未找到 package.json | ZIP 根目录缺少 `package.json` |
| package.json 中未找到 jsos 配置 | 缺少 `jsos` 字段 |
| 清单缺少必要字段 | `id` / `name` / `startCommand` 缺失 |
| 不支持网络图标 | `icon` 使用了 HTTP 链接（请用 data URI） |
| 图标文件不存在 | `icon` 指定的文件不在 ZIP 包中 |

## 安装后

- 应用图标出现在**当前工作区桌面**
- 点击图标即可启动
- 可以在**应用管理器**中查看已安装应用列表

## 卸载应用

右键应用图标 → **「卸载」**，确认后应用被移除。可选是否同时删除应用数据。

> **系统应用**（预装应用）不可卸载。
